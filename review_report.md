## 一、严重问题（实现描述与代码不符，建议必改）

### 1. 数据集来源表述与实现不一致（且自相矛盾）
- **报告**：摘要/1.2/3.2 称训练用 `nlile/24-game (solvable=True) 1262`；幻觉集写 `nlile/24-game (solvable=False)` 又写"合成无解四元组"。
- **不自洽/不实**：① 实际 `game24.py` 只 `load_dataset("test-time-compute/game-of-24")`，**train/test 同源按 Rank 切分**（train Rank 1–1262 / test Rank 1263–1362），**全程未加载 nlile**；② 幻觉集"nlile solvable=False"与"合成无解四元组"互斥，实际是**合成**（`{1..13}⁴` 减可解集，已 `solve24()` 验证 100 题全无解，代码注释明确"nlile 已无 solvable=False 行"）；③ 3.2 把同一来源拆成两个来源。
- **实际信息**：改为 train/test 均来自 test-time-compute、幻觉集为合成。（两数据集题目相同，结果不受影响，但出处必须改对。）

### 2. 推理引擎与硬件与实际不符（"声称做了但没做"）
- **报告**：3.1 "vLLM 作为推理引擎"、"硬件：单卡 RTX 3060 12GB"。
- **不实**：本环境 vLLM ABI 损坏不可用，实际用 **HF rollout**（`rollout.name=hf`，并为此给 `hf_rollout.py` 打 n>1 补丁、加 `VERL_DISABLE_FLASH_ATTN_CE` 开关）；硬件是 **8×A100-80GB**（隔离 Ray 并行多 arm）。3060/vLLM 是参考 README 的设定。
- **实际信息**：rollout=HF、GPU=8×A100；这是本项目最主要的工程改动，应写进去。

### 3. 超参表与实跑/正文步数不一致
- **报告**：3.3 表 `Batch=32 / Max Resp=512 / Epochs=15`；4.1 又说"训练 160 步"。
- **不自洽**：`32×15×(1262/32)≈590 步`≠160 步。实跑是 `batch=16 / max_response=256(v1)320(v2) / total_epochs=3≈160 步`（off-by-one 实存到 `global_step_120`），lr 1e-6、KL 0.001、ent 0.001、T 1.0。`32/512/15` 是参考脚本默认值。
- **实际信息**：超参表改成实跑值。

### 4.【重点·你指出的】"SFT v1 / v2"（及一堆内部标签）全程没定义——要么解释清楚，要么别写出来
- **报告**：摘要"对 **SFT v2** 模型…"；3.3 Base Model 写"**SFT v2** checkpoint"；4.1"从基座和 **SFT v1** 出发"；4.2"以 **SFT v2** 为统一起点"；表1"SFT-only **(v1)**"。

## 三、更深的逻辑不自洽 / 缺内容（你强调的，最该补）

### 9.【核心矛盾】"推荐 SFT→GRPO" 与 "GRPO 让 best-of-N/多样性退化" 两个结论互相打架，未调和
- **报告**：5.1(1) 把 **SFT→GRPO** 当作主推荐；5.1(3)/4.6 又说 **GRPO 训练使策略熵坍缩、best-of-N 随步数下降**；5.1(4) 还说"未来方向不在于学更多，而在更稳定访问已有知识"，而 50% 天花板是 **SFT v2（GRPO 之前）** 的。
- **不自洽**：既然 GRPO 压低了 best-of-N，而 best-of-N 天花板恰恰来自 GRPO **之前** 的 SFT v2，那"做 GRPO"到底是帮忙还是帮倒忙？报告没回答。**实测佐证**（`results/ablation_summary.md`）：`sftv2base bv@16=21% → control40=12% → control80=5%`——**GRPO 步数越多、best-of-N 越差**。
- **应补**：明确把 GRPO 的作用写成一个**权衡（trade-off）**：GRPO 抬高 greedy/in-dist（27→42%）、但压低 best-of-N/多样性；据此给出"何时该停 GRPO/早停"的建议，而不是笼统"SFT→GRPO 最好"。这其实是本实验最有价值的发现，却被两处割裂叙述了。

### 10. 实验设计表（3.3）与实际报告的实验臂对不上 + 缺 SFT 超参
- **报告**：3.3 只列了 `Phase-1` 与 `Wave-1 (Control)` 两列；但正文实际报告了 **n8 / shaped / staged-p1**（4.2）和 **DivA–DivD**（4.3/4.6）等多臂，3.3 完全没涵盖；且 3.3 **只有 GRPO 超参，没有任何 SFT 超参**（lr/epochs/数据量/轨迹构造）。
- **缺内容**：读者无法从"配置表"对上后面出现的实验臂；SFT 作为核心手段却没有自己的超参/数据表。**应补**：把 n8/shaped/staged/DivA–D 的配置补进表（n、KL、ent、T、reward、步数），并单列一张 SFT 配置表。

### 11. 只给了 SFT v2 的 best-of-N 天花板，缺 GRPO 各臂的天花板对比
- **报告**：4.4 只测了 **SFT v2** 的 best-of-N（k 到 64、不同 T，最高 50%）。
- **缺内容/逻辑缺口**：没有给 GRPO 各臂（control/n8/shaped/Div*）在 **同样 k=64** 下的天花板，因此无法回答"**GRPO 把天花板抬高了还是压低了**"。结合第 9 条（GRPO 降 best-of-N），很可能 GRPO 各臂的 k=64 天花板 < 50%，但报告没给这条关键对比，导致"50% 潜力"与"该不该 GRPO"脱节。

### 12. 统计噪声未处理：难测试 n=100，1% 就是 1 道题
- **报告**：在难测试集上比较 0% / 1% 等差异（如 Pure GRPO 1% vs n8 0%），并据此措辞。
- **逻辑/严谨性问题**：测试集只有 **100 题**，1% = **1 道题**；表2 里 0%↔1%↔2% 的差异**在统计噪声范围内**，不足以支撑"谁更好"的结论。**应补**：注明 n=100（1%=1题）、对关键对比给多种子/多次采样或置信区间，难测试这类近似 0 的指标改用 **best-of-N / 分布内 / Countdown(256)** 这些方差更小的量来下结论。

### 13. "纯 GRPO 几乎无法从零学习"措辞过强
- **报告**：5.1(1) "纯 GRPO 几乎无法从零开始学习"。
- **轻微不自洽**：表1 显示纯 GRPO **确实学到一点**（in-dist 0%→5%、Countdown 1.95%→2.34%、训练奖励末段从 ~0.13 爬到 ~0.2）。应改为"纯 GRPO 学习极慢/严重受限"，与数据一致。

### 14.【逻辑矛盾】KL 散度的解读前后打架
- **报告**：4.1 "SFT→GRPO 的 KL 升到 0.24、纯 GRPO 0.14，**说明 SFT 后能有更大的策略更新空间**"（把 KL **高** 当好事）；4.3/DivC 又 "通过**加强 KL 惩罚把策略锚定在参考模型附近**"（把 KL **低** 当好事）。
- **问题**：同一个量，一处"越高越好"、一处"越低越好"，未给统一口径。建议统一为"KL 是更新幅度/偏离参考的度量，过低=学不动、过高=不稳，需折中"。

### 15.【自破规矩】shaped 的 0.54 与 sparse 的 0.109 被并列
- **报告**：2.2 明确"塑形奖励的验证集得分与稀疏奖励**不可直接比较**"；4.2 却"staged-p1 val/test_score=**0.54**，control 仅 **0.109**"并排放。
- **问题**：这正是它自己说不能比的两个量（0.54 含"接近 24"部分分，**不是准确率**）。应删除该并列，或仅用表2的离线 sparse 贪心对比。（另见"准确率 vs 平均奖励"口径，见 `补充材料.md` §6.0。）

### 16.【提到却未定义】DivA / DivB 是"幽灵"
- **报告**：5.2(5) 写"DivA–DivD 仅训练 20–40 步"，但 4.3 只介绍了 **Control / DivC / DivD**；**DivA、DivB 全文无定义、无图表**。实际 divA = n4/KL0.005/ent0.01/T1.2、divB = n8/KL0.01/ent0.02/T1.2（见 `补充材料.md` §4.3）。要么补全，要么不写"DivA–DivD"。

### 17.【指标退化】幻觉"0% 编造率"近乎必然，不能作为"不瞎编"的证据
- **报告**：4.1/表1/5.1 以"所有模型幻觉 0%"论证"奖励数字校验有效、模型不瞎编"。
- **问题**：幻觉集是**无解题**，**根本不存在**用给定 4 数 = 24 的算式，故"求解率（score==1.0）"**结构上恒为 0**，对任何模型（含 base）都是 0% —— 近乎同义反复，**不构成"不瞎编"的证据**。真正的幻觉测试应衡量：模型是否**自信给出声称=24 但实则不等于的算式**、或**是否会判定"无解/放弃"**。当前指标测不出这些。

## 四、已核实属实、**不是问题**（供放心，勿误删）
- "基模 1%、n=4 每组至少一条正确 ≈4%"：`1-0.99⁴≈3.9%`，自洽。
- "T=1.5、k=64 达 50%"**有据**：`results/ceiling/T1.5.log` `bv@64=0.5`（T1.0=37%、T1.2=39%），可标注出处 `results/ceiling/`。

- 幻觉"求解率 0%"这个**数字**本身没错（与代码一致）；但它作为"不瞎编"的**论据**无效（见第 17 条），二者不冲突。
- **准确率口径正确**：`greedy_acc / bv@k / n_correct` 均按 `score==1.0`（真正 =24）计，非把 0.1 格式分算进去；需要澄清的只是别把"平均奖励/val_test_score"（含 0.1）当成准确率（见 `补充材料.md` §6.0）。
